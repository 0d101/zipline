zipline.utils subpackage¶

zipline.utils.__init__.run_pipeline(print_algo=True, **kwargs)¶

Runs a full zipline pipeline given configuration keyword arguments.

1. Load data (start and end dates can be provided a strings as well as the source and symobls).

2. Instantiate algorithm (supply either algo_text or algofile kwargs containing initialize() and handle_data() functions). If algofile is supplied, will try to look for algofile_analyze.py and append it.

  1. Run algorithm (supply capital_base as float).
  2. Return performance dataframe.
Arguments:
  • print_algo : bool <default=True>

    Whether to print the algorithm to command line. Will use pygments syntax coloring if pygments is found.
zipline.utils.__init__.parse_args(argv, ipython_mode=False)¶

Parse list of arguments.

If a config file is provided (via -c), it will read in the supplied options and overwrite any global defaults.

All other directly supplied arguments will overwrite the config file settings.

Arguments:
  • argv : list of strings

    List of arguments, e.g. [‘-c’, ‘my.conf’]

  • ipython_mode : bool <default=True>

    Whether to parse IPython specific arguments like –local_namespace

Notes: Default settings can be found in zipline.utils.cli.DEFAULTS.

zipline.utils.factory.load_from_yahoo(indexes=None, stocks=None, start=None, end=None, adjusted=True)¶

Loads price data from Yahoo into a dataframe for each of the indicated securities. By default, ‘price’ is taken from Yahoo’s ‘Adjusted Close’, which removes the impact of splits and dividends. If the argument ‘adjusted’ is False, then the non-adjusted ‘close’ field is used instead.

Parameters:
  • indexes (dict) – Financial indexes to load.
  • stocks (list) – Stock closing prices to load.
  • start (datetime) – Retrieve prices from start date on.
  • end (datetime) – Retrieve prices until end date.
  • adjusted (bool) – Adjust the price for splits and dividends.
zipline.utils.factory.load_bars_from_yahoo(indexes=None, stocks=None, start=None, end=None, adjusted=True)¶

Loads data from Yahoo into a panel with the following column names for each indicated security:

  • open
  • high
  • low
  • close
  • volume
  • price

Note that ‘price’ is Yahoo’s ‘Adjusted Close’, which removes the impact of splits and dividends. If the argument ‘adjusted’ is True, then the open, high, low, and close values are adjusted as well.

Parameters:
  • indexes (dict) – Financial indexes to load.
  • stocks (list) – Stock closing prices to load.
  • start (datetime) – Retrieve prices from start date on.
  • end (datetime) – Retrieve prices until end date.
  • adjusted (bool) – Adjust open/high/low/close for splits and dividends. The ‘price’ field is always adjusted.

simfactory Module¶

zipline.utils.simfactory.create_test_zipline(**config)[source]¶
Parameters:config –

A configuration object that is a dict with:

  • sid - an integer, which will be used as the security ID.
  • order_count - the number of orders the test algo will place, defaults to 100
  • order_amount - the number of shares per order, defaults to 100
  • trade_count - the number of trades to simulate, defaults to 101 to ensure all orders are processed.
  • algorithm - optional parameter providing an algorithm. defaults to zipline.test.algorithms.TestAlgorithm
  • trade_source - optional parameter to specify trades, if present. If not present zipline.sources.SpecificEquityTrades is the source, with daily frequency in trades.
  • slippage: optional parameter that configures the zipline.gens.tradingsimulation.TransactionSimulator. Expects an object with a simulate mehod, such as zipline.gens.tradingsimulation.FixedSlippage. zipline.finance.trading
  • transforms: optional parameter that provides a list of StatefulTransform objects.
